Find The Mole

Find The Mole

Find the mole is a real time chat based game.

The game was developed using .NET8 for the backend and Angular 17 for the frontend.

SignalR was used for real time synchronization and message exchange.

The rules of the game are quiet simple. A game has a number of players varying from three to ten.

When a game starts, a location among 15 possible locations is chosen and one of the players will be the Mole while the others will be detectives.

The detectives will know what the location is and will have to guess who the mole is by asking other players questions. When a detective thinks he knows who the mole is, he can request to vote.

As for the mole, he will have to try and figure out what the game location is and at any point in the game, he can make a guess. When he does the game will immediately be over and he will win if the guess was correct or lose otherwise. Like the other players, the mole can request a vote. This strategy can be used when he feels that the group is more suspicious of another player.

If the number of the players who want to vote exceeds half of the initial number of the players, all the players are redirected to the vote screen where they have to vote for who they think the mole is. Once all the players have voted, the game is over. If the mole gets the majority of the votes, the group wins otherwise the mole wins.

Dys E-Learning Platform

Dys E-Learning Platform

Huruf Toon is user-friendly web application tailored specifically to Tunisian learners and to the local education system. This adaptive platform aims to address the shortcomings of existing solutions by providing customized exercises for different types of dys disorders. This was a group project in our second year and was selected in the annual Bal De Projet organized by ESPRIT as one of the best projects.

To develop our platform, we created several machine learning and deep learning models and deployed in a web application using the Flask framework for backend development alongside Jinja 2 as template engine.

Our platform has three main features:

Level assessment quiz and dyslexia type identification: To create the quiz, we created two models using machine learning classifiers ( After comparing several models we ended up using AdaBoost in both models) for the detection of Dyscalculia and Dysgraphia. As for Dyslexia, we used a speech-to-text to assess the child's reading ability and then we used the Levenshtein Distance Algorithm to see how accurate his reading was.

Personalized and Adaptive Training Activities: During this phase, we created several types of exercises, using several machine learning and deep learning ( mainly CNNs ) models to boost the child's ability and performance on various cognitive skills such as reading, writing, counting...
We also tried to boost the child's artistic capabilities by allowing him to draw pictures and then comparing them to a prompt. To achieve this end, we used YOLOv8 to detect the drawn objects.

Development of Dyslexia Chatbot: We also crafted a chatbot that answers to dys related questions employing two models BERT and GPTLMHead. While BERT excels in understanding contextual nuances, GPTLMHead shines in generating coherent and contextually relevant responses.

IFRS 9 Dashboard

IFRS 9 Dashboard

After the Subprime Crisis on 2008, the International Accounting Standards Board adopted IFRS 9 which is an International Financial Reporting Standard that addresses the accounting for financial instruments, regulating credit risk management to avoid future crisis.

In this project, we developed a dashboard that could be used by financial institutions, to monitor loans using the IFRS 9 Standard. Our dashboard provides a well-structured view, starting with an overview of the credit portfolio and progressively drilling down into specific details, such as segmentations by credit types, regions...
It also allows access to detailed information on individual creditors' credit data

To achieve this, we first modeled a Data warehouse and then using SSIS , we extracted the data from the excel files provided to us and loaded them into a Staging Database .

Using the tools offered by SSIS , we cleaned the data, made the required transformations and then loaded it into the Data warehouse . After that, we created a SSAS cube which we used as our data source in Power BI .

In order to deploy our dashboard, we created a secure web application using Spring Boot for backend development and Angular for the frontend.

In this app we used JSON Web Tokens to handle authentication. We further secured the passwords using the b-crypt algorithm. And finally, we used role-based access control for authorization.
